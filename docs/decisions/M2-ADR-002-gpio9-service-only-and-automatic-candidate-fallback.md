# M2-ADR-002：GPIO9 仅限服务写入与候选身份自动回退

- **状态：** 已批准
- **批准日期：** 2026-07-19
- **批准方：** 产品负责人
- **适用范围：** ESP32-C6 产品固件、M2 节点身份迁移和实板故障矩阵

## 决策

1. GPIO9 只用于研发、生产和授权售后进入 ESP32-C6 ROM USB 下载模式。
2. 产品固件不得把 GPIO9 注册为按键、传感器、回滚输入或其他运行期功能。
3. 用户 OTA 故障流程不得要求拆机、接触 PCB、按 GPIO9、复位或自行 USB 写入。
4. 未提交 candidate 使用自动安全机制：首次候选启动写入持久化标记；10 分钟租约到期自动选择 anonymous 并安全重启；提交前再次启动则在 MQTT 初始化前直接恢复 anonymous。
5. M2 矩阵以 `rollback.candidate_lease_expired` 替换 `rollback.offline_button`。旧项目私有证据不得改写为新用例已通过；新用例必须重新取得实板证据。

## OTA 与 USB 恢复边界

产品目标是 OTA 新镜像首启失败时自动恢复上一可用镜像。该能力必须同时具备并验证：双 OTA 分区、ESP-IDF bootloader rollback、明确的首启本地健康检查、成功后标记镜像有效，以及断电/崩溃/外设异常的 A/B 实板矩阵。

当前 M2 身份回退不等同于固件镜像 OTA 回滚，也不得据此宣称 OTA 自动回滚已经完成。上述 OTA 门完成前，USB ROM 重写仅作为工厂或授权售后的最后救援方式，由受训人员拆机操作；它不是用户功能，也不写入用户操作指南。

## 安全边界

- 不擦除 NVS；
- 不关闭 anonymous；
- 不访问或修改生产系统；
- 不生成或下发生产节点凭据；
- 候选身份回退不得擦除 candidate secret 或冒充安全擦除；
- 编译、仿真或合同测试不能替代新矩阵用例的实板证据。
