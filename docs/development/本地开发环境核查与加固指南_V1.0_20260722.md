# 温室环境监测系统：本地开发环境核查与加固指南

**版本：** V1.0  
**日期：** 2026-07-22  
**基线：** `main=b8cc4f68d29393cdf9da7d00fdfeef28ee147c7b`  
**开发分支：** `chore/local-dev-environment-hardening-20260722-v1`  
**工作分类：** B 类开发工具与本地安全边界  
**默认状态：** 只读、无网络探测、无设备访问、无生产环境访问、无系统修改

## 1. 目标

本阶段不重新搭建开发环境，也不复制 T1、M401A 或 Home Assistant 生产栈。目标是把现有 Mac 开发机固化为可重复核查、可审计、默认安全的本地工作站：

1. 统一仓库、虚拟环境和证据目录；
2. 检查 Git、Python、ESPHome、pytest、Ruff、Docker 和 GitHub CLI 状态；
3. 检查错误远程仓库、受保护分支开发、脏工作区和敏感文件误跟踪；
4. 检查本地密钥、密码文件和证据目录权限；
5. 提供显式授权的加固工具；
6. 提供只检查暂存区的本地提交保护；
7. 生成不含秘密值的 JSON 审计证据。

## 2. 权威路径

默认策略固定以下路径：

```text
权威开发仓库：/Users/chenrenguo/HomeAssistant-local-test
Python 虚拟环境：/Users/chenrenguo/.venvs/greenhouse-homeassistant-dev
外部证据目录：/Users/chenrenguo/HomeAssistant-evidence
```

历史目录 `/Users/chenrenguo/HomeAssistant` 不作为新开发入口，不自动删除，也不由本工具修改。

## 3. 工具组成

| 文件 | 作用 | 默认是否修改环境 |
|---|---|---:|
| `tools/dev/local_environment_policy_20260722_v1.json` | 路径、版本、安全和执行边界策略 | 否 |
| `tools/dev/local_environment_doctor_20260722_v1.py` | 只读核查并生成脱敏报告 | 否 |
| `tools/dev/apply_local_environment_hardening_20260722_v1.py` | 创建私有证据目录、写本地非秘密配置、可选安装 Hook 和收紧文件权限 | 仅 `--apply` 后 |
| `tools/dev/hooks/pre_commit_local_environment_guard_20260722_v1.py` | 阻止暂存运行时状态、凭据文件和私钥内容 | 安装后仅阻止提交，不改文件 |
| `tests/tools/test_local_environment_tooling_20260722_v1.py` | 回归测试 | 否 |

所有 Python 工具仅依赖标准库。加固工具不会安装软件、修改全局 Git 配置、启动 Docker、连接 Broker、访问串口或操作实机。

## 4. 分阶段执行

### L0：隔离取得工具分支

为避免打断正在进行的产品开发分支，优先创建独立 worktree：

```bash
cd /Users/chenrenguo/HomeAssistant-local-test

git fetch origin \
  chore/local-dev-environment-hardening-20260722-v1

git worktree add \
  /Users/chenrenguo/HomeAssistant-local-env-hardening-20260722 \
  origin/chore/local-dev-environment-hardening-20260722-v1
```

该操作不会切换权威开发仓库当前分支。

### L1：只读审计

```bash
TOOL_ROOT=/Users/chenrenguo/HomeAssistant-local-env-hardening-20260722
REPO=/Users/chenrenguo/HomeAssistant-local-test
VENV=/Users/chenrenguo/.venvs/greenhouse-homeassistant-dev
EVIDENCE=/Users/chenrenguo/HomeAssistant-evidence
REPORT_DIR="$(mktemp -d /tmp/gh-local-env-audit.XXXXXX)"

"$VENV/bin/python" \
  "$TOOL_ROOT/tools/dev/local_environment_doctor_20260722_v1.py" \
  --policy "$TOOL_ROOT/tools/dev/local_environment_policy_20260722_v1.json" \
  --repo "$REPO" \
  --venv "$VENV" \
  --evidence-dir "$EVIDENCE" \
  --json-output "$REPORT_DIR/local-environment-audit.json"

AUDIT_RC=$?
echo "AUDIT_RC=$AUDIT_RC"
echo "AUDIT_REPORT=$REPORT_DIR/local-environment-audit.json"
shasum -a 256 "$REPORT_DIR/local-environment-audit.json"
```

含义：

- `AUDIT_RC=0`：没有失败项，可能仍有警告；
- `AUDIT_RC=2`：存在必须处理的失败项；
- 报告不包含密码、Token、密钥或环境变量值；
- 默认不会执行 GitHub 登录、网络、Docker daemon、SSH、MQTT 或设备探测。

首轮只需回传：完整终端输出、`AUDIT_RC`、报告 SHA-256 和 JSON 报告。

### L2：加固计划预演

审计结果确认后，先执行 dry-run：

```bash
"$VENV/bin/python" \
  "$TOOL_ROOT/tools/dev/apply_local_environment_hardening_20260722_v1.py" \
  --repo "$REPO" \
  --venv "$VENV" \
  --evidence-dir "$EVIDENCE" \
  --install-pre-commit-hook \
  --restrict-sensitive-files
```

预期必须出现：

```text
MODE=DRY_RUN
RESULT=dry_run_complete
No filesystem or Git hook changes were made.
```

### L3：显式应用基础加固

只有 dry-run 计划与审计结果一致时才执行：

```bash
"$VENV/bin/python" \
  "$TOOL_ROOT/tools/dev/apply_local_environment_hardening_20260722_v1.py" \
  --apply \
  --repo "$REPO" \
  --venv "$VENV" \
  --evidence-dir "$EVIDENCE" \
  --install-pre-commit-hook \
  --restrict-sensitive-files
```

该命令只会：

1. 创建或收紧 `~/HomeAssistant-evidence` 为 `0700`；
2. 创建或收紧 `~/.config/greenhouse-dev` 为 `0700`；
3. 写入不含秘密值的本地路径配置，权限为 `0600`；
4. 将发现的本地敏感文件权限收紧为 `0600`，明确示例文件除外；
5. 安装本地 pre-commit guard；如已有不同 Hook，先生成带时间戳的备份；
6. 自动运行一次只读复核，并将报告写入外部证据目录。

### L4：工具链版本冻结

本策略当前要求：

```text
Python：>=3.11.0 且 <3.12.0
ESPHome：2026.4.3
pytest：8.x
Ruff：0.15.22
```

版本不一致时，审计只报告，不自动升级或降级。依赖调整必须在审计后单独形成安装计划，避免破坏现有可工作的 ESP-IDF、PlatformIO 或 ESPHome 缓存。

### L5：接入本地 Preflight

本阶段完成后，再把 doctor 纳入统一 `gh-local preflight`。在此之前，不修改现有 `gh-local fast` 行为，也不让环境审计自动触发固件编译、Docker 构建或生产环境检查。

## 5. 审计项目

doctor 当前检查：

- macOS 与 CPU 架构；
- 可用磁盘空间，最低 15 GiB、建议 30 GiB；
- Git、SSH、SCP、SHA-256 工具；
- 可选 GitHub CLI、Docker、ESPHome、PlatformIO；
- 权威虚拟环境及 Python 版本；
- ESPHome、pytest、Ruff 版本；
- Git 顶层目录、当前分支、HEAD、工作区状态和 origin；
- 是否直接在 `main` 开发；
- 是否跟踪 `.env`、`secrets.yaml`、`.storage`、`.esphome`、数据库、私钥和证据目录；
- 暂存区是否包含私钥标记；
- 本地敏感文件是否存在过宽权限；
- 外部证据目录是否位于仓库外且权限为 `0700`；
- 是否存在已赋值的敏感环境变量名称，仅报告变量名，不读取或输出值。

## 6. Pre-commit guard 边界

Hook 只检查暂存区：

- 禁止提交 `.env`、`secrets.yaml`、`credentials.json`、数据库、私钥文件和 Home Assistant/ESPHome 运行时状态；
- 扫描文本中的私钥头；
- 明确允许 `.env.example` 和 `*.example.pem`；
- 不联网、不读取未暂存文件、不修改 index；
- 大于 2 MiB 或二进制文件不做内容扫描，但路径规则仍生效。

不应长期使用 `git commit --no-verify` 绕过保护。确有安全示例文件需求时，应先调整规则并补测试。

## 7. 回滚

加固不会删除原文件。回滚方法：

1. Hook 安装时如发现旧 Hook，会打印 `BACKUP=...`；需要恢复时将备份复制回 `pre-commit`；
2. 可删除 `~/.config/greenhouse-dev/local-environment-v1.json`，其中不含秘密；
3. 证据目录不建议删除；若确需清理，先归档审计报告；
4. 文件权限从 `0600` 放宽必须由用户明确决定，本工具不提供自动放宽；
5. 工具 worktree 可在阶段结束后执行：

```bash
cd /Users/chenrenguo/HomeAssistant-local-test
git worktree remove /Users/chenrenguo/HomeAssistant-local-env-hardening-20260722
```

## 8. 验收标准

本地环境加固完成需同时满足：

- doctor `FAIL=0`；
- origin 指向 `chrenguo-stack/HomeAssistant`；
- 权威仓库与虚拟环境路径正确；
- Python、ESPHome、pytest、Ruff 版本符合策略或存在已批准豁免；
- 外部证据目录位于仓库外且权限为 `0700`；
- 不存在被 Git 跟踪的真实凭据、私钥或运行时状态；
- pre-commit guard 能阻止测试私钥并允许普通文本；
- 加固后报告 SHA-256 已保存；
- 未连接真实 Broker，未操作 NVS、eFuse、ESP32-C6、M401A、T1 或生产容器。

## 9. 当前阶段的停止条件

出现以下情况立即停止，不执行 `--apply`：

- 权威仓库存在尚未确认的本地修改；
- origin 不正确；
- 虚拟环境路径缺失或 Python 主版本不符；
- 发现真实私钥或凭据已被 Git 跟踪；
- 证据目录位于仓库内部；
- 已有 pre-commit Hook 且其职责不清楚；
- 审计输出意外出现秘密值；
- 任一命令试图访问生产主机、Broker 或物理设备。
