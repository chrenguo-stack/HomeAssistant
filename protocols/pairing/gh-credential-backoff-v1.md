# gh-credential-backoff-v1 错误凭据退避

状态：Draft / M2.3b  
关联：Issue #17、`gh-pairing-v1`

## 1. 规则

MQTT Broker 明确拒绝凭据时，节点按以下基础秒数重试：

`1, 2, 4, 8, 15, 30, 60, 60, ...`

每次增加 ±20% 随机抖动，实际延迟不得低于 1 秒。一次成功认证将失败计数清零。普通网络不可达不增加“凭据拒绝”计数，由网络连接层独立处理。

## 2. 本地功能隔离

- 传感器采集、LCD、阈值判断和本地告警不得等待 MQTT 重连；
- 错误凭据不得触发重启循环；
- 错误凭据不得自动清除 owner、Wi-Fi 或 active generation；
- 错误凭据不得静默打开配对窗口；
- 只有显式用户操作、已认证 manager 命令或恢复出厂可以进入重新配对。

## 3. 诊断与秘密

允许记录错误码、失败计数和下一次重试的相对秒数；禁止记录 username、password、Broker 控制响应或完整凭据 bundle。失败计数只用于退避，不作为撤销或恢复出厂依据。

## 4. 本阶段验收

M2.3b 先在节点模拟器中冻结确定性策略并验证上限、抖动、成功复位、网络故障隔离和本地采集不受影响。ESP32-C6 固件接入另行实机验收。
